Introduction[edit]

This article details the instructions for generating CPT statistical throughput data for CorePac0 accesses to CorePac1's L2 memory space. In this example a program running on CorePac0 is generating memory accesses to CorePac1 and CorePac1's CPT module will be utilized to measure the performance of those accesses. The CP Tracer STM data is collected by the ETB, thus this example can be integrated into your application to remotely collect data. This example uses CCS just to load the application example and upload the CP Tracer STM trace data from the ETB to files on your host (through CIO).   CCS will also be used to display the data (see Results section).

Note: The same instructions below can be used to load and run the System Profile example for your specific device. In this example rather than examining data generated by a single CP Tracer, all CP Tracers are enabled but only the bandwidth data for each is provided. In the instructions below simply replace the cpt_l2_etb_d_corepac0.xxxx.out file with the cTools\Examples\bin\cpt_sp_etb_d_corepacN.xxxx.out.

Requirements[edit]

See Common Platform Tracer Examples for CCS and target requirements. Any XDS may be used with this example.

Download[edit]

  • Download the latest Examples.zip from ctoolsLib-gforge. It's recommended that you create a "cTools" directory and expand the zip file to this directory.
  • Optional step. If you need to rebuild the example, you will also need the latest CPTLib.zip and STMLib.zip, both of which are available from ctoolsLib-gforge. Unzipped these to the "cTools" directory created for the previous step.

Instructions[edit]

  1. Start CCS and launch your target configuration
  2. From the Debug window highlight all cores, then right click and select Connect Target. There is no need to connect CS_DAP_DebugSS, CSSTM_0 or the ETB for this example.
  3. Run any GEL scripts necessary to initialize your target. If you have already setup your CCS configuration to automatically launch a gel that initializes your target skip to the next step. If not, for eample in the case of the TMS320TCI6616 EVM, load the EVMTCI6616.gel file and select one of the scripts to set the functional clock rate of the device Scripts=>EVMTCI6616 Init Functions=>Set_Pll1_xxx_MHz. To load the gel file select the processor in the Debug Window and right click to bring up a menu from which you select Open Gel Files View.  
  4. To load the target application select  Target=>Load Program and then load:
    For TMS320C667x and TMS320TCI6616 cTools\Examples\bin\cpt_l2_etb_d_corepac0.6670.out
    For TMS320C6614/2 cTools\Examples\bin\cpt_l2_etb_d_corepac0.tci6614.out
    For TMS320C6657 cTools\Examples\bin\cpt_l2_etb_d_corepac0.c6657.out
    Note that this example is built for generating data for the Total Bandwidth Profile use case. To change the use case see notes below.
  5. With CorePac0 (C66XX_0) selected in the Debug view hit the Run button
  6. When execution is complete the trace data is uploaded to two files (c:\temp\CPT_etbdata.bin and c:\temp\CPT_dcm.txt). This may take a few seconds.
  7. The CorePac0 (C66XX_0) CIO console will show any errors and the number of bytes uploaded to the CPT_etbdata.bin file.

Note: The use case for the example can be changed by modifying the following macros in Examples\C667x\CPT_L2_CorePac0\CPT_L2_Example.c and rebuilding the project:

#define TOTAL_BANDWIDTH_PROFILING 1

#define MASTER_BANDWIDTH_PROFILING 0

#define EVENT_USE_CASE 0

It is required to only run one profile use case per recording session.

Results[edit]

To see a the CP Tracer profile data in a CCS Trace Display window:

  1. Open a command window at ccsv4\emulation\analysis\bin and execute the following:
    >bin2tdf -bin c:\temp\CPT_etbdata.bin -app C66X_0=c:\cTools\Examples\bin\cpt_l2_etb_d_corepac0.6670.out -devicekey 0x0009D02F -procid stm -rcvr ETB -dcmfile c:\temp\CPT_etbdata.dcm -output c:\temp\CPT_etbdata.tdf
    Note The out file and -devicekey will need to be modified for your specific target. 0x0009D02F is the device ID for C667x and TCI6616. 0x0b94102F is the device id for TCI6618. 0x0B96202F is the device ID for TCI6614 and TCI6612. 0xB97A02F is the device id for C6657/5/4.
  2. In CCS select Tools=>Trace Analyzer=>Open Trace File in New View...
  3. CCS will request the file to open. You provide the c:\temp\CPT_etbdata.tdf file generated by bin2tdf.
  4. CCS may also request the program file
  5. Double left click on the STM Trace Displayy tab to expand the window to the full Eclipse window, double clicking again on the tab will restore the window size.

To see a the CP Tracer profile data plotted graphically by CCS:

  1. Left Click on the Trace Display window tab (highlighting the tab)
  2. Select Tools=>Trace Analyzer=>Analysis=>STM Analysis=>STM Analysis Graph
  3. Double left click on the STM Analysis Graph tab to expand the window to the full Eclipse window, double clicking again on the tab will restore the window size.
CPTLib ETB CorePac1L2 TotalUseCaseGraph.png


  • Red - MB/s of CorePac0 accesses to CorePac1 L2 memory
  • Purple - Total Transactions for all masters accessing CorePac1 L2*
  • Green - Average Access width in bytes
  • Turquoise - % of CorePac1 L2 bandwidth used by CorePac0 (measured against all other active bus masters, not max bandwidth). Note that in this example the percentage of bandwidth used by CorePac0 is 100% because CorePac0 is the only active master accessing the CorePac1 L2.
  • Pink - Overall Average latency
