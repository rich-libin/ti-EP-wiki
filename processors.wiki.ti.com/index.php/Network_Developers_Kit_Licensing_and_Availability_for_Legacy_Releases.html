How much does NDK cost?[edit]

The NDK is free to download. Please see download links provided in the questions that follow.

Is there support for my application development?[edit]

You can submit issues to TI's e2e support forum, however these are restricted to the core stack, drivers and examples running on our development boards.

The main reason is because it is virtually impossible to support every combination of client configuration or network topologies available in the marketplace.

Also, to help your development, the latest NDK ships with full source code. This comprises all the example applications, the device drivers and the NDK core.

Additional support information can be found throughout this wiki. Please go to Main Page on the left menu and check the Network Developer's Kit category.

Is my board supported? Where is the software located?[edit]

The latest NDK release notes provide a detailed list of the processors supported.

MCU device support is provided by the TI-RTOS product (see the TI-RTOS Download Page and release notes for details).

C6000 multicore device support is provided in the MCSDK product (see the MCSDK landing page and release notes for details).

For legacy boards and releases supported please see the NDK Product Matrix.PDF, as well as the NDK 2.0.0 release notes.

Does each board have a specific release? Or is there a common version of NDK for all the boards?[edit]

The NDK core stack is completely abstracted from the hardware and common for all supported ARM and DSP processors. The latest releases ship with full source code and makefiles to allow custom (re)building of the NDK libraries. Please refer to the NDK release notes (linked on the download page for each NDK release) for a complete list of processors supported.

In addition to the core stack, another piece of software is also needed: the EMAC device driver package, called the NDK Support Package (NSP). The NSPs contain software that is specific to a particular hardware platform. This includes the Hardware Abstraction Layer (HAL), the low-level device drivers and the example CCS projects for the evaluation boards. Typically, there are different NSP products for different hardware platforms and it is required in order to build an NDK application.

Some hardware platforms also require peripheral drivers, which are found in a product called the Platform Support Package (PSP).

The NSPs and PSPs are provided 'as-is' and any changes and/or adaptations to custom hardware are unsupported (it's impossible to support every single combination of EMAC/PHYs in the marketplace).

How can I get the NDK?[edit]

The latest NDK releases can be found on the NDK Download Page.

Older NDK releases can be found on the Update Advisor website (Target Content Updates -> select Network Developer’s Kit in the menu on the left).

How can I get the NSP?[edit]

The latest NSP releases for the OMAPL138 can be found on the NDK Download Page.

MCU device support is provided by the TI-RTOS product and can be found on the TI-RTOS Download Page.

C6000 multicore device support is provided in the MCSDK product and can be found on the MCSDK Landing Page.

Older NSP releases (which support other hardware platforms) can be found on the Update Advisor website (Target Content Updates -> select Network Developer’s Kit in the menu on the left).

NSP support for other hardware platforms should be included with the SDK that comes with your hardware package.

How can I get the PSP?[edit]

The latest PSP releases can be found on this page.

Older PSP releases can be found on the Update Advisor under DSP/BIOS and Target Content Updates -> Processor Support Packages (PSP) -> DSP/BIOS-based Platform Support
