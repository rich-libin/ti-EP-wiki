Release Notes

Release Date:  November 30, 2012

Introduction[edit]

This is Release 04.06.00.09 of AM335x Linux PSP. This release supports:

  • BeagleBoard Bone (version A3 and above)
  • AM335x Evaluation Module (version 1.1A and above)
  • AM335x Evaluation Module-StarterKit (version 1.2B and above)

The Linux PSP serves to provide a fundamental software platform for development, deployment and execution of Linux based applications on Texas Instruments silicon.

The Linux kernel and U-Boot software in this release are based on the following open source repositories:

What's Supported:[edit]

Refer to the What's Supported section from AM335x-PSP 04.06.00.08 Release Notes for a complete list of What's Supported in this PSP.


New in this Release:[edit]

  • RTC Runtime PM Support
  • TSC/ADC: IIO and MFD framework migration
  • ADC driver support
  • tps65217: Set PMIC to shutdown using RTC module
  • USB PHY Wakeup support

    Note: Since in DS0 usb wakeup can not be supported on current silicon, by default USB wakeup is kept disabled.

  • CPSW: Promiscuous mode support
  • CPSW: Support the HWTSTAMP ioctl and the CPTS


Fixed in this Release:[edit]

  • PWM: ecap: Resets the PWM output to low on stop
  • net: ethernet: fix kernel OOPS when remove davinci_mdio module
  • SDOCM00094923: NFS mount fails with "DHCP/BOOTP: ignoring delayed packet" messages
  • mtd: nand: omap: fix race condition in omap_wait()
  • mtd: nand: omap2: Fix for uncorrectable error
  • mtd: nand: omap2: Fix bit flip in data area due to bit flip in OOB
  • mtd: nand: omap2: Fix ECC read for 16-bit NAND part
  • ARM: OMAP2+: am335x: Fix xbar channel implementation for odd channels
  • SDOCM00085443 - ARM: OMAP2+: am335x: Change the wdt1 func clk src to per_32k clk
  • SDOCM00086716 - Fix omap_hwmod wdt2: _wait_target_disable failed error message
  • ARM: OMAP: AM33XX: PM: Fix potential deadlock in resume process due to MPU leaving GPIO0 enabled
  • USB:
  • Build warning fixes
  • usb: musb: Fixes for invalid parameter passed for dev_dbg()
  • SDOCM00097396 - Babble workaround fix
  • SDOCM00097395 - Rx race condition
  • SDOCM00091956 - The debugfs support for usb testmode
  • SDOCM00091464 - The UVC camera doesn't capture frames after several times of off and on
  • SDOCM00095817 - AM335x: CPPI DMA RX does not handle the dummy byte in ZLP
