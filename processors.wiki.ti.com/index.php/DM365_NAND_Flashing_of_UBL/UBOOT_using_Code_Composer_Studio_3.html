Purpose of this page[edit]

  • In order to Flash UBL/UBOOT binary files to a DM365 EVM with the NAND flashing utility, it is very important for the correct software versions of Code Composer Studio, Emulation Software Drivers and the CCS setup driver be installed nn your PC. The following steps below will help align the correct software PC tools for burning UBL and UBOOT files on a DM365 EVM



CCS version : 3.3.38.13 (Service Release 12 or Newer version patches)[edit]

  • External web installation location: http://software-dl.ti.com/dsps/dsps_registered_sw/sdo_ccstudio/CCSv3/SR/SR12_CCS_v3.3_SR_3.3.82.13.exe


Figure shows how to verify that you have the correct CCS version:

Spectrum Digital XDS510 USB emulation drivers version: 3.03.29 or newer versions[edit]

Note: Only one version of the SD configuration drivers is needed, having multiple versions of Spectrum Digital drivers will create issues.

  • It is recommended that ONLY the latest Spectrum Digital SD config drivers are installed for XDS510 USB emulator. This can be verified in PC control panel-> add/remove programs-> SD CCS 3.3-> "click here for support info"
  • Spectrum Digital installation location: http://support.spectrumdigital.com/ccs33/


CCS driver configuration for DM365 using Spectrum Digital's XDS510 USB JTAG Emulator[edit]

  • Spectrum Digital's location: http://support.spectrumdigital.com/boards/evmdm365/revc/files/DM365-XDS510USB.ccs

CCS Gel file for DM365 using Spectrum Digital's XDS510 USB JTAG Emulator[edit]

  • Download the gel file from (for Rev C or later): http://support.spectrumdigital.com/boards/evmdm365/revc/
  • Copy the gel file (optional) to C:\CCStudio_v3.3\boards\evmdm365_v1\gel

Testing CCS-JTAG-EVM Interface[edit]

  • Make sure that when connecting the USBXDS510 emulator to the DM36x EVM you can double check that you have JTAG communication by running the SDCONFIG utility(v3.3) that gets installed during step #2 above (as shown in picture below). From this utility run "configuration->verify" and you will see the enumeration behavior as shown in the picture and once it completes it will also show a message that says:

  • The CCS NAND programmer and NAND erraser .out files can be found inside the following website: http://dleinspect.itg.ti.com:82/dsps/dsps_public_sw/sdo_sb/targetcontent/dvsdk/DVSDK_3_10/latest/exports/board-utils-davinci.tar.gz (untar it and the NAND Programmer(NANDWriter_DM36x.out) and NAND eraser(NANDEraser_DM36x.out) are under the \board_utilities\serial_flash\dm365\ directory)

Flashing the DM365 EVM[edit]

Erasing the current Flash[edit]

  • Initial DM365 EVM SW4/SW5 pin setting:
    • EMIF BOOT(SW4 =000000)- selects ROM NAND BOOT mode
    • EMIF BOOT(SW5 =100010)- selects OneNAND Chip Select Space and 1.35V core voltage Mode. (This prevents previously flashed Linux Kernel from booting from NAND)
  • There is a difference between REV B and REV C (or later) of the DM365 EVM. In REV B EVM PIN 4 and 5 of SW5 should be set 1. In REV C EVM only PIN 5 of SW5 should be set to 1




  • Ensure that the EVM GEL file is setup with your CCS (e.g 'evmdm365.gel' for DM365) this is typically provided by Spectrum Digital on the specific EVM website or it comes with your CCS installation.
  • Connect CCS and allow for the Gel file to complete initializing your device.
  • In CCS load the NANDeraser_DM36x.out into RAM
  • In DM365 EVM change the SW4/SW5 pin setting after NAND eraser .out file has been loaded and leave this pin setting for the remaining NAND flashing utility steps.
    • EMIF BOOT(SW4 =000000)- selects ROM NAND BOOT mode
    • EMIF BOOT(SW5 =000010)- selects NAND Chip Select Space.



  • Execute the NAND eraser (CCS Debug->Run). The NAND Eraser will ask for an input where user is asked to input the location where the NAND is connected. Input '0x02000000'. This will erase the lower half of the NAND. To erase the upper half execute the program again and input '0x02004000'. NANDs comes in various flavours. We can have a 2GB NAND where it is internally 2 devices of 1 Gb each and there are others where internally the NAND is actually 1 device of 2GB. In the latter case we do not have to execute the program twice.
  • The NAND eraser does not need to be used twice in the case where the 2GB SLC NAND from SAMSUNG(K9KAG08U0M) with page size 4K is used.

Loading the NAND Flash Writer Program[edit]

  • In CCS load the NANDwriter_DM36x.out into RAM and execute the programmer (CCS Debug->Run)
    • The CCS Standard Input Dialog Box will pop up and the NAND programmer will then ask for the UBL that needs to be written to NAND. Give UBL_xx.xx.bin along with the entire path where the binary is located (e.g ...\..\..\UBL_DM36x_NAND.bin )
    • The CCS Standard Input Dialog Box will pop up again an then Input the U-Boot binary. Input u-boot-xx-xx-xx.bin along with the entire patch where the binary is located.(e.g ...\..\..\u-boot-1.3.4-dm365_evm.bin )
    • The CCS Standard Input Dialog Box will pop up again and the final two inputs required by the NAND writer are the U-Boot entry point and the load address. Both of these should be 0x81080000.
    • You have completed the flashing of UBL/UBOOT successfuly if the CCS Stdout window displays a message "NAND boot preparation was successful!"
    • You can now Disconnect CCS
  • The NAND Programmer has a feature where we can choose not to flash the UBL or U-Boot. For example if we want to just change the UBL, we can choose to burn the new UBL and then when the NAND programmer asks for the U-Boot binary, we can choose none. Similarly, if we want to just change the U-boot, then when the NAND programmer asks for the UBL binary, we can choose none and input the new U-Boot binary when prompted for it.

Verifying that the Flashing was done[edit]

  • If you have not already done so connect a NULL modem cable from DB9 connector on EVM to COM port on PC. Then program the COM port per the instructions in the DM365 DVEVM Hardware Setup
  • Power up the EVM and you should see the U-Boot prompt on the serial console (e.g. TeraTerm Pro).
