Introduction[edit]

Trace Control is used to configure either the device's internal ETB or an XDS560v2 System Trace for STM collection.

ETB Instructions[edit]

To select and setup the ETB for STM collection use the following steps:

  1. Select Tools=>Trace Control to bring up the Trace System Control window.
  2. Use the tab navigation arrows (top right corner) to navigate to the "XDS name/CSSTM_0" tab
  3. If Receiver is not selected (tab page indicates Receiver needs to be selected)) or the ETB is not selected then hit the Receiver button and select "ETB"
  4. ETB type should be set to "Auto".
  5. The Sample Configuration's "Synchronize trace with target execution" box should be checked. This option stops the trace system from recoding trace data when the processor is halted, and automatically starts recoding when processor execution is started.
  6. Trace Buffer Type should be set to "Circular buffer".
  7. Hit the OK button to complete setup.

XDS560v2 Instructions[edit]

To select and setup the XDS560v2 System Trace for STM collection use the following steps:

  1. Select Tools=>Trace Control to bring up the Trace System Control window.
  2. Use the tab navigation arrows (top right corner) to navigate to the "XDS name/CSSTM_0" tab
  3. If Receiver not selected (tab page indicates Receiver needs to be selected) then hit the Receiver button and select "560V2Trace"
  4. Set Pin Configuration Trace Bits to "4-Bits" (See Note 1 below for more details)
  5. Set Receiver Configuration DTC Buffer Size to "256K". (See Note 2 below for more details)
  6. The Sample Configuration's "Synchronize trace with target execution" box should be checked. This option stops the trace system from recoding trace data when the processor is halted, and automatically starts recoding when processor execution is started.
  7. Set Trace Buffer Type to "Circular Buffer". (See Note 3 below)
  8. Hit the OK button to complete setup and calibrate.

If no errors are reported the Trace Control window is closed automatically.

Notes:

  1. Selecting 4-bits will provide the maximum bandwidth. If you know that your XDS connection does not support EMU[0:5] then you may need to select "2-pin" EMU[0:2] or "1-pin" EMU[1:0].
  2. You can select a larger buffer, but keep in mind a larger buffer will require additional decode time.
  3. A "Circular Buffer" will over write older data with newer data providing visibility from the end of your application backwards. In this case you may need to expand the buffer to capture earlier data. "Stop on Buffer Full" will show you data from the time you started collecting data until the buffer is full. Keep in mind that CP Tracers will start generating data as soon as they are enabled unless qualified with a trigger-in. If using "Stop on Buffer Full" you may need to expand the size of the trace buffer to collect all the applications trace data.
