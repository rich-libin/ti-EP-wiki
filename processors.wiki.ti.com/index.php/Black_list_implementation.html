Black list implementation

From Texas Instruments Wiki
Jump to: navigation, search

When security is enabled, black list of unwanted networks is useful for a joining device to avoid re-attempting to join the networks where it has already failed to be authenticated.

What is Black List?[edit]

Black List is a list of the networks that a joining device seeking a proper secured network needs to avoid attempting to join.

Why is Black List necessary?[edit]

Authentication failure[edit]

In Standard Security Mode, the most popular security mode in ZigBee applications, a joining device is notified of being authenticated by receiving a network key from its parent through Transport Key message. ZigBee specification describes this procedure as following, in section 4.6.3.2.2.1:

If the joining device did not receive the key via the APSME-TRANSPORT-KEY.indication within the apsSecurityTimeOutPeriod since receiving the NLME-JOIN.confirm primitive, it shall reset and may choose to start the joining procedure again.

What's happening after reset[edit]

"Reset" in Z-Stack is the same as power-cycle. If a joining device is reset, it is supposed to do the joining process from scratch including network discovery, parent candidates selection, and attempt to join one candidate. The list of parent candidate consists of the best router/coordinator in each pair of channel and PANID available in range in terms of the signal strength. Of course, only the routers/coordinators that meet conditions like the minimum RSSI, the maximum depth, and association permit status are considered. Once the list is established, the joining device picks the first candidate whose PANID and extended PANID match the desired ones if specified. Protocol version, router capacity, and end device capacity are also checked. Assuming desired PANID and extended PANID are not specified, there always is a possibility that the candidate which the joining device has already failed in authentication procedure with comes first in the list. If it happens, the joining device will attempt to join that parent and fail to be authenticated again and reset itself.

How Black List filtering is implemented[edit]

The Black List implementation is composed of two major parts: one for adding unwanted network to the list and the other for filtering out the listed network.

Adding the PANID of a failed network to the Black List[edit]

If a joining device doesn't receive the network key from the parent within apsSecurityTimeOutPeriod which is set to 10 seconds by default in Z-Stack, ZDApp_event_loop() will receive ZDO_DEVICE_RESET event and the global variable devState will still have the state DEV_END_DEVICE_UNAUTH at that moment. Before resetting, the PANID of the network the parent belongs to is stored in the Black List located in the NV items area reserved for applications.

Ignoring beacons from the networks on the Black List[edit]

At Network Discovery stage, a joining device receives beacons from the routers/coordinators in range. Before adding a router's or a coordinator's PANID and network address to the parent candidate list, the Black List is checked. If the PANID is found in the Black List, it is not added to the parent candidate list.

  1. ifdef BLACK_LIST

/********************************************************************* 

* @fn      zgBlackListInit()
*
* @brief   Initialize the black list, the list is read from NV
*          when needed.  If you want to initialize the black list with
*          some specific PANIDs, do it here.
*
* @param   setDefault - TRUE to set default
*
* @return  ZSUCCESS if successful, NV_ITEM_UNINIT if item did not
*          exist in NV, NV_OPER_FAILED if failure.
*/

uint8 zgBlackListInit( uint8 setDefault ) { 

 zgBlackList_t blacklist;
 uint8 status;

 //  Initialize the black list, the list is read from NV
 //  when needed.  If you want to initialize the black list with
 //  some specific PANIDs, do it here.
 osal_memset( &blacklist, 0xff, sizeof(zgBlackList_t) );
 blacklist.numItem = 0;

 status = osal_nv_item_init( ZCD_NV_BLACK_LIST, sizeof(zgBlackList_t), &blacklist );
 if ( status == ZSUCCESS )
 {
   if ( setDefault )
   {
     // Write the default value back to NV
     status = osal_nv_write( ZCD_NV_BLACK_LIST, 0, sizeof(zgBlackList_t), &blacklist );
   }
 }

 return status;

}

/********************************************************************* 

* @fn       zgBlackListAdd()
*
* @brief
*
*   Add a PANID to the black list. If the PANID already exists in
*   NV memory, return ZSUCCESS without adding it. If there is no room
*   for the new PANID in NV memory, return ZBufferFull. Otherwise
*   perform osal_nv_write() to add the PANID.
*
* @param   panid - PANID to be added to the black list
*
* @return  ZSUCCESS if successful, ZBufferFull if there's no room for
*          panie, return value of osal_nv_write() otherwise.
*/

uint8 zgBlackListAdd( uint16 panid ) { 

 uint8 status, i;

 zgBlackList_t blackList;

 status = osal_nv_read( ZCD_NV_BLACK_LIST, 0, sizeof(zgBlackList_t), (void*) &blackList );
 if ( status == ZSUCCESS )
 {
   // if there is no room for panid, return ZBufferFull
   if ( blackList.numItem >= MAX_BLACK_LIST_ITEM)
   {
     return ZBufferFull;
   }

   // if the panid already exists in NV memory, return ZSUCCESS
   for ( i = 0; i < blackList.numItem; i++ )
   {
     if ( blackList.panid[i] == panid )
     {
       return ZSUCCESS;
     }
   }

   blackList.panid[blackList.numItem++] = panid;
   status = osal_nv_write( ZCD_NV_BLACK_LIST, 0, sizeof(zgBlackList_t), (void*) &blackList );
 }

 return (status);

}

  1. endif // BLACK_LIST

</syntaxhighlight>
