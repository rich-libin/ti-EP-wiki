Working with A band (5Ghz band)[edit]

In order to work with A band follow the steps below:[edit]

  1. 5GHz band is supported by WL1273 chipset (WL1271 supports only 2.4GHz band)
  2. Modify the fast clock to work with WL1273 module (fast clock configuration is implemented in kernel)
  3. Define the regulatory Domain to support A band Domain (for example US)
  4. Perform Calibration with dual band ini file, to support 2.4GHz band and 5GHz band


Detailed Instructions[edit]

  • Supporting fast clock configuration (that requires building the kernel):
  • AM18x platform: wl1273 support (for 802.1A band)
  • AM37x platform: wl1273 support (for 802.1A band)
  • AM335x platform: wl1273 support (for 802.1A band)


  • Regulatory Domain
  • How to control Regulatory Domain
  • WLAN rates
  • WLAN channels


  • Calibration – Calibration is a one-time procedure, performed before any WLAN operation. Calibration is performed once after the board assembly, or in case the 1271 connectivity daughtercard or EVM are replaced

Note: for supporting A band use the TQS_D_1.7.ini in the Calibration process (TQS_S_2.6.ini is used for WL1271 chipset)

Verify the port configuration:[edit]

In order to verify that the port support both 2.4Ghz (bg band) and 5Ghz (A band), need to run the command: iw phy0 info
The expected results is that both 2.4Ghz and 5Ghz are supported by the port.
