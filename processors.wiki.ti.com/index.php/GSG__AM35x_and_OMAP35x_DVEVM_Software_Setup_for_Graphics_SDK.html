Requirements[edit]

The following are the requirements for software setup:

  • PC Linux Host
    • Root login or liberal sudo privileges
    • Most Linux Distributions should work with this Graphics SDK
    • This Graphics SDK has been tested on
      • Red Hat Enterprise Linux, Workstation V4, x86 32 bit
  • Linux Support Package(PSP) for AM35x and OMAP35x EVM and 387x/389x – This graphics SDK package has been built and tested with TI PSP GIT release version as specified in the readme.pdf file
  • File system for SDK.
  • Toolchain version as specified in readme.pdf.
  • On 387x/389x, there is a dependancy of fbdev on syslink and IPC modules. Please refer to the readme.pdf to use the exact syslink and IPC versions required.

Please visit the below link that provides the readme.pdf for all the Graphics SDK versions released so far- http://processors.wiki.ti.com/index.php/OMAP35x_Graphics_SDK_Release_Notes_Archive

Preparing to Install[edit]

Download the following installers to the temp folder /tmp from the OMAP3 software update site as mentioned here

  • OMAP35x-PSP-SDK-setuplinux-##.##.##.##.bin
  • Graphics_SDK_setuplinux_#_##_##_##.bin

The # refers to the version numbers
The AM35x/AM37x/OMAP35x/387x/389x Graphics SDK contains the pre-built graphics drivers and kernel modules that are already built with the appropriate version of the OMAP35x/37x/389x/387x Linux PSP. Make sure that this dependency is maintained and the user uses the kernel image built from the correct version of the AM35x/AM37x/OMAP35x and 387x/389x SDK as mentioned in readme.pdf

Ensure all the .bin files are set with executable permissions. 

host $ chmod 777 *.bin


Installing the Software[edit]

Installing the software used by the AM35x/OMAP35x EVM and 387x/389x involves performing the following steps.

  1. Installing the Target Linux Software
  2. Installing the Graphics SDK Software
  3. Installing the toolchain


Installing the Target Linux Software[edit]

Refer OMAP35x DVEVM Getting Started Guide: Installing_the_Target_Linux_Software

Installing the Graphics SDK Software[edit]

To install the Graphics SDK software using the Linux installer, follow these steps:

  1. Log in using a user account. In the following steps, we refer to the home directory as "~".
  2. Execute the Graphics SDK installer that you previously downloaded from the extranet location. For example: 
    host $ ./Graphics_SDK_setuplinux_#_##_##_##.bin

    The installer will start as a GUI application. Follow the instructions in the dialog boxes. You’ll be asked to agree to the End User License Agreement.

    When you are prompted for an installation location, use the default installation location, that points to /home/<user_account>/Graphics_SDK_#.##.##.##. This location will be used as the Graphics SDK installation folder through out this document.

  3. You can now delete the .bin files that you loaded into the temporary directory.

Note: You can uninstall these components by using the uninstall file in the respective installation directories. For example: 

host $ cd /home/<user_account>/OMAP35x_Graphics_SDK_#_##_##_##
host $ ./uninstall


Installing the Toolchain[edit]

This guide assumes use of the LITE version of the CodeSourcery toolchain. See CodeSourcery documentation if you are using the evaluation version that comes with this EVM or for other cases. The toolchain used is ARM GNU/Linux EABI 2009q1. It can be downloaded via the following URL: http://www.codesourcery.com/sgpp/lite/arm/portal/release858 Select : IA32 GNU/Linux TAR

Alternative method is as below:

http://www.codesourcery.com/sgpp/lite/arm/releases/2009q1

Select the following options: Target Platform: ARM GNU/Linux Host Platform: IA32 GNU/Linux

To install the toolchain, follow the sequence below. These are to be executed on the Linux host platform. 

$ mkdir –p /home/<user_account>/toolchain
$ cp arm-2009q1-203-arm-none-linux-gnueabi-i686-pc-linux-gnu.tar.bz2 /home/<user_account>/toolchain
$ cd /home/<user_account>/toolchain
$ tar -jxvf  arm-2009q1-126-arm-none-linux-gnueabi-i686-pc-linux-gnu.tar.bz2
$ rm arm-2009q1-203-arm-none-linux-gnueabi-i686-pc-linux-gnu.tar.bz2


Setting up the Build/Development Environment[edit]

To set up the development and build environment, follow these steps:

  1. Log in to your user account (and not as root) on the Linux host system, where the AM35x/OMAP35x or 387x/389x Graphics SDK is installed.
  2. Add the /host/<user_account>/toolchain/arm-2009q1/bin directory to your path. This is typically done by adding an additional line to your shell resource file (~/.bashrc). For the path given above, the line to add to your .bashrc file is: 
    PATH="/home/<user_account>/toolchain/arm-2009q1/bin:$PATH"

    This adds the CodeSourcery tools to your path and allows you to execute the tools using arm-none-linux-gnueabi-gcc (or other tools in the tool chain) from any directory.

  3. Remember to use the following command after modifying your .bashrc file: 
    host $ source .bashrc
  4. You can test that the toolchain is installed correctly by starting a new shell and using the following command: 
    host $ arm-none-linux-gnueabi-gcc –v
