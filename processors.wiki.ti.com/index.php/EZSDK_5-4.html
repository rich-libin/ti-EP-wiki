Component patches that are present in the EZSDK[edit]

The EZSDK comes with a set of pre-applied patches to several software components. These patches have been applied to fix issues discovered during EZSDK software validation, or update the components to match EZSDK configurations of memory.

All patches can be found in <software component directory>/patches

Software Component Patches Remarks
EDMA3 LLD Patch support for sdk.mk This patch adds support for install into devkits.
Graphics SDK Patch support for sdk.mk This patch adds support for install into devkits.
Linuxutils Patch support for sdk.mk This patch adds support for install into devkits.
OSAL Patch support for sdk.mk This patch adds support for install into devkits.
U-Boot Patches to support a Boot Logo This patch adds support for a splashscreen in u-boot
Linux Kernel (DM814x) Patches to support WLAN and Crypto drivers This patchset adds support for WLAN and Crypto drivers.
GStreamer Patches to GStreamer, GST-Plugins Number of patches have been applied to GStreamer and different GST Plugins. These improve GStreamer performance on DM81xx. For more details please see GStreamer TI

EZSDK 5.04.00.11 Errata[edit]

This list contains a list of issues that are present in the EZSDK release. This list includes the issues noted in the release notes and other issues reported post-release.

  1. SDOCM00073168 - SDK installer deletes the entire contents of the install folder when installation is cancelled (Cloned from DVSDK 4.0).
  2. SDOCM00085969 - Syslink Sample applications do not run out of the box. Workaround - The BIOS dmtimer has to be reconfigured for 20MHz. The samples need to be rebuilt and run.
  3. SDOCM00085970 - Codec Engine Example applications do not run out of the box. Workaround - The BIOS dmtimer has to be reconfigured for 20MHz. The examples need to be rebuilt and run.
  4. SDOCM00086122 - QT Multi: Driver does not repaint all regions. As a result when using multiple graphics planes, only one will be refreshed. No Workaround exists.
  5. SDOCM00087096 - Unable to display on some TVs at 1080i. Workaround none exist.

Download the Latest EZSDK[edit]

The latest EZSDK is available for download from http://software-dl.ti.com/dsps/dsps_public_sw/ezsdk/latest/index_FDS.html.

The current version is 5.05.02.00. The supported platforms are DM816x and DM814x.
