Before you begin[edit]

To begin developing applications, you need to install the EVM development environment. This section outlines the steps required to load the EVM software onto the development host.

  • Follow the steps in the kit's Read Me 1st Card to register and download the most current software from TI's website. (If you have already registered this EVM, skip to the next step.)
  • Log on to myRegistered Software, where you can download the TI software you will need.

Installing the software[edit]

  1. Find the following Files for software installation:

    You will need to install each of the following files (or equivalent versions):
    • C6748_dsp_setupwin32_1_00_00_04.exe: This is an installer for PC workstations and will install the same contents as installing OMAPL138_dsp_setupwin32_1_00_00_04.exe.
    • bios_setupwin32_5_33_06.exe: This the installers for PC workstations and will install the DSP/BIOS software.
    • setupwin32_ndk-2_0_0_0.exe: This is an installer for PC workstations and will install the evaluation version of the Network Development Kit (NDK) software running on C6748 DSP.
    • TI_CGT_C6000_6.1.9_setup.exe: This is the installers for PC workstations and will install the TI code generation tool. It is needed for users to who do not have the up-to-date code generation tool. If using the CCS that comes with the Starter Kit you do not need to install this because it already comes with Code Generation Tools 6.1.5.
    • xdctools_setupwin32_3_15_01_59.exe: This is the installers for PC workstations and will install the XDC tool. XDC tool is needed to build various software including Codec Engine, Framework Components etc
    NOTE: We strongly suggest installing the software below to the default installation paths. Deviating from the default paths may cause problems with building and is not covered in this guide.

  2. The Zoom OMAP-L138 EVM Development Kit includes a software CD containing Code Composer Studio, as well as other EVM tools. This software allows for interaction with the OMAP-L138 EVM and helps with diagnostics.
    • Note: The software CD is not included with OMAP-L138 eXperimenter Kits. Instead, the software is available as an .iso download from the Logic website at the following URL (requires product registration): http://support.logicpd.com/downloads/1232/
  3. Code Composer Studio is included on the software CD. Insert the “OMAP-L138 / TMS320C6748 Code Composer Studio v3.3 IDE” software CD into a development PC.
  4. A splash screen similar to the image below will appear. Click on the “Install Software” button.
  5. Click on the “Code Composer Studio” button.
  6. Follow the on screen instructions. When you arrive at the screen for type of installation, select “Typical Install.” This will install both Code Composer Studio and the DSP BIOS.
  7. Continue with the on screen instructions. After you click the “Install Now” button, wait for Code Composer Studio to finish installing; this process can take some time, so please be patient.
  8. Click the “Finish” button when the installation is complete.

Installing EVM Drivers and BSL[edit]

  1. On the “Install Software” splash screen for the software CD, click the “EVM Drivers and BSL” button.
  2. Follow the on screen instructions to install the EVM drivers and Board Support Library (BSL) sample programs.
  3. Click the “Finish” button when the installation is complete.
  4. On the “Install Software” splash screen, click the “Main Menu” button and then the “Exit” button to close out the software CD installation program. You may eject the software CD at this time.
  5. To know if you need to install the Code Generation Tools (TI_CGT_C6000_6.1.9_setup.exe file) and DSP/BIOS (bios_setupwin32_5_33_06.exe file), open CCS, go to the menu Help -> About. Make sure that you have Code Generation Tools 6.1.5 or above and BIOS 5.33.03 or above. If you do not have the required versions, install the correspondent file (TI_CGT_C6000_6.1.9_setup.exeand/or bios_setupwin32_5_33_06.exe).
  6. After Installing DSP/BIOS 5.33.06 (if you did not already have it), open CCS, go to the menu Help -> About. Click on Component Manager. The component manager screen will open. Expand the Target Content (DSP/BIOS) list. Expand the lists for TMS320C64XX and TMS320C67XX and select Texas Instruments DSP/BIOS 5.33.06 on both lists. Go to the menu, File -> Save to save the configuration. Restart CCS for the changes to take effect.
  7. After Installing Code Generation Tools 6.1.9 (if you did not already have it), open CCS, go to the menu Help -> About. Click on Component Manager. The component manager screen will open. Expand the Build Tools list. Expand the lists for TMS320C64XX and TMS320C67XX and select Texas Instruments C6000 Code Generation Tools <6.1.9> on both lists. Go to the menu, File -> Save to save the configuration. Restart CCS for the changes to take effect.

Installing PSP (Platform Support Package)[edit]

  1. Install the file C6748_dsp_setupwin32_1_00_00_4.exe. Once you have installed the C674x_dsp_setupwin32_1_00_00_10.exe you must also install the BIOSPSP_01_30_00_05_Setup.exe in the base install directory (C:\C6748_dsp_1_00_00_4 by default). This file will install the DSP/BIOS device drivers for the C6748 (the default directory is C:\Program Files\Texas Instruments\pspdrivers_01_30_00_5). To get the stand alone version of the BIOSPSP go to this page (you will need a ti.com ID).

  2. Install the XDC tool by installing the file xdctools_setupwin32_3_15_01_59.exe.

  3. If you want to work with networking, install the NDK from the file setupwin32_ndk-2_0_0_0.exe and also the Network Support Driver from the file ndk2.0_omapl138_nsp_01_00_00_01.exe.

  4. If you want to work with USB, install the file BIOSUSB_01_00_00_Setup.exe. For the latest stand alone version please see this page.

  5. Install the real-time file system for DSP/BIOS (RTFS). To check for updates to RTFS, please go to this page. You will need a ti.com ID to login.
