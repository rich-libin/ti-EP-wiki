Field Translator[edit]

Overview[edit]

The Field Translator is a powerful component that can be used for numberous string and numberic decoding and operations. It transforms data according what is specified in an XML file. Functionality includes:

  • Rename/change format/delete/add columns of data
  • perform string manipulation
  • numeric calculation
  • bitwise operation
  • conditional operation
  • delta and accumulate operations

Input Channel[edit]

1 input channel

Output Channel[edit]

1 output channel

Properties[edit]

Property Description Type Default
Name Displays the unique name for the component. Name can be changed through GSB Text
XMLconfig The XML file in which the required translations are specified File Dialog

Operations[edit]

passthru[edit]

The <passthru /> tag is used to specify a field that will appear on the output channel. This tag is generally used for passing through fields that are received on the input channel but it also used to rename or manipulate the field (using an expression). Note that if a passthru tag has not been specified for an input field, the respective column will be filtered out and shall not appear on the output channel.

Below is the list of attributes for the passthru

Attribute Description
name the name of the output field the data will be passed to
input an expression of the input fields that will be passed to the output
filterBlanks Skips records that have blank values for this field(disabled by Default). Set value to "true" to enable 
  E.g. Passthrough input field named "xyz" as is.
       <passthru name="xyz" input="xyz" />

  E.g. Rename input field "xyz" to "abc"
       <passthru name="abc" input="xyz" />

  E.g. Manipulate input field "xyz"
       <passthru name="xyz" input="xyz" + 25 />

  E.g. Passthrough input field named "xyz" as is and remove records which have blanks for field "xyz"
       <passthru name="xyz" input="xyz" filterBlanks="true"/>

passthruAll[edit]

The <passthruAll /> tag is used to pass all the fields from the input to the output without having to list them individually using the <passthru /> tag. There is also an easy way to remove certain fields from passing thru.

There are no attributes for the passthruAll tag

passthruAll can also be used in combination with the following lower level tag:


  • <removeField/>

And has the following attributes

Attribute Description
name specifies the name of the field not to be passed thru 


  E.g. Passthrough all fileds from input except field "xyz".
       <passthruAll>
            <removeField name="xyz" />
       </passthruAll>

stringParser[edit]

The <stringParser /> tag is used to divide text strings into one or more numerical or text-based output columns.

Attribute Description
input The input field
delimiters charecter(s) to be used to seperate the text into fields
multi treats blank spaces as fields
pattern
translate
filterBlanks Skipes blank records (disabled by Default)


stringParser can also be used in combination with the following lower level tags:


  • <string />
  • <integer />
  • <float />
  • <hexubyte />
  • <hexchar />
  • <hexushort />
  • <hexuint />
  • <hexulong />
  • <hexubytearray />
  • <timestamp /> Not supported

Each of the above tags have the following attributes that are applicable

Attribute Description
name specifies the name of the output field
translate a string consisting of token numbers or regular expression groupings 
 <string /> tag in addition has attribute map which is optional. This is a comma 
 separated string of key value pairs to map a string value to another string value example



     Example of string parser using delimiters attribute
     <stringParser delimiters=" " input="Line">
        <integer name="Time1" translate="$1" />
        <integer name="Priority1" translate="$2" />
        <string name="Event1" translate="$3" />
        <integer name="Size1" translate="$4" />
    </stringParser>

     Example of string parser using regular expression in the pattern attribute
    <stringParser pattern="(\d+) +(\d) +(\w+) +(\d+)" input="Line">
        <integer name="Time2" translate="$1" />
        <integer name="Priority2" translate="$2" />
        <string name="Event2" translate="$3" />
        <integer name="Size2" translate="$4" />
    </stringParser>

recordEval[edit]

The <recordEval /> tag is used to perform record based evaluation. This tag can also be used in combination with the following lower level tag: <item />

Below is the list of attributes for the recordEval

Attribute Description
field Output field name
type Output field data type can be of the following
auto string integer uinteger ulong
defaultval The default output field value
display The display format, can be one of the following
Decimal Hex Scientific Binary
filterBlanks Skips blank records(disabled by Default)


Below is the list of attributes for the <item /> tag

Attribute Description
value (optional) The output value when the evaluated expression is "true". The value can be:
absolute, E.g. "12"
copy from an in field E.g. "@time"
expression A regular expression or one of the 2 special forms:
“delta(_inField_): find the delta value of the specified in field. Delta = current value – previous value
“accum(_inField_): the the accumulative value of the specified in field 


 <recordEval field="fixed" type="integer" display="Decimal">
   <item expression="12"/>
 </recordEval>
 
 <recordEval field="ReadCmdPri" type="auto" defaultVal="0">
   <item value="@Priority" expression="Event == "ReadCmd"" />
 </recordEval>
 
 <recordEval field="TotalSize" type="integer" display="Decimal">
   <item expression="accum(Size)"/>
 </recordEval>
 
 <recordEval field="DeltaTime" type="integer" display="Decimal">
   <item expression="delta(Timestamp)"/>
 </recordEval>

renameAll[edit]

Not supported

transpose[edit]

The <transpose /> tag creates an output field that substitutes textual strings for specified numerical input values. This tag can also be used in combination with the following lower level tag: <label /> The label tag specifies a name value pair when substituting text for numerical values.

Below is the list of attributes for the transpose

Attribute Description
name the name of the output field for the transposed data
input an expression of the input fields that will be passed to the output
filterBlanks Skipes blank records (disabled by Default) 
 E.g.
 <transpose name="Words"            input="Numbers"> 
   <label  name="ONE" value="1" />	  
   <label  name="TWO" value="2" />	  
 </transpose>

symbolic[edit]

Not supported

In this example we will do the following

  • Parse text field using delimiters
  • Parse text field using regular expression
  • For both of the above we will form the various output fields


 <translate name="translate">
     <stringParser delimiters=" " input="Line">
        <integer name="Time1" translate="$1" />
        <integer name="Priority1" translate="$2" />
        <string name="Event1" translate="$3" />
        <integer name="Size1" translate="$4" />
    </stringParser>
    
    <stringParser pattern="(\d+) +(\d) +(\w+) +(\d+)" input="Line">
        <integer name="Time2" translate="$1" />
        <integer name="Priority2" translate="$2" />
        <string name="Event2" translate="$3" />
        <integer name="Size2" translate="$4" />
    </stringParser>  
 </translate>

Case 2: Basic Fields Manipulation (Rename, Add, Delete, Modify, Retain)[edit]

  • Input

1200 2 WriteCmd 8
1300 3 WriteCmd 8

In this example we will do the following

  • Rename Field Timestamp to Time
  • Retain field Event and it values
  • Modify values of field Size by multiplying it by 4
  • Add a field called Urgency
  • Delete field Priority
 <translate name="translate">
     <passthru name="Time"                     input="Timestamp" /> 
     <passthru name="Urgency"                  input="Priority < 2 ? "HIGH" : Priority==2 ? "MEDIUM" : "LOW"" /> 
     <passthru name="Event"                    input="Event" /> 
     <passthru name="Size"                     input="Size*4" />
 </translate>

Case 3: Refering to values from Previous records[edit]

  • Input

Same as input used in Case 2

In this example we will do the following

  • Create a new field called ConsecWrite which will have value YES if previous record and current record have Event field value as WriteCmd
  • Create a new field called PrevValidSize which will have the value of the last valid Size value from previous records. Note invalid numeric field value would be if it is marked as NaN (i.e. not a number)
 <translate name="translate">
     <passthru name="Time"                     input="Timestamp" /> 
     <passthru name="Priority"                 input="Priority" /> 
     <passthru name="Event"                    input="Event" /> 
     <passthru name="ConsecWrite"              input="((Event$1 == "WriteCmd") && (Event == "WriteCmd")) ? "YES" : """ />     
     <passthru name="Size"                     input="Size" />
     <passthru name="PrevValidSize"            input="Size$valid" />     
 </translate>
  • Output
Time Priority Event ConsecWrite Size PrevValidSize
1000 0 ReadCmd 8
1100 0 ReadResp 8 8
1200 2 WriteCmd 8 8
1300 3 WriteCmd YES 8 8
1400 1 ReadCmd 22 8
1500 0 WriteCmd 8 22
1600 1 ReadResp 16 8

Case 4: transpose[edit]

  • Input

Same as input used in Case 2

In this example we will do the following

  • Transpose the values in Size field from a numeric to a string and having value as the number spelt out. We will create a new field called TransposeWORDSize
  • We will only provide transposes for some of the possible values, the ones which do not have one specified will just have the number present as string
 <passthru name="Time"            input="Timestamp" /> 
 <passthru name="Priority"        input="Priority < 2 ? "HIGH" : Priority==2 ? "MEDIUM" : "LOW"" /> 
 <passthru name="Event"           input="Event" /> 
 <passthru name="Size"            input="Size" /> 
 
 <translate name="translate">
     <transpose name="TransposedWORDSize"            input="Size"> 
         <label  name="EIGHT" value="8" />	 
         <label  name="SIXTEEN" value="16" />	  
         <label  name="TWENTY_FOUR" value="24" />	  	  
     </transpose>  
 </translate>
  • Output
Time Priority Event Size TransposedWORDSize
1000 0 ReadCmd 8 EIGHT
1100 0 ReadResp 8 EIGHT
1200 2 WriteCmd 8 EIGHT
1300 3 WriteCmd 8 EIGHT
1400 1 ReadCmd 22 22
1500 0 WriteCmd 8 EIGHT
1600 1 ReadResp 16 SIXTEEN

Case 5: recordEval[edit]

  • Input

Same as input used in Case 2

In this example we will do the following

  • accumulate the value of Size in a field called TotalSize
  • present the delta between Timestamp field (current - previous) in a field called DeltaTime
  • Create a new field called ReadCmdPri which will have the value of the Priority field when the record is for a ReadCmd Event.
 <translate name="translate">
     <passthru name="Time"                     input="Timestamp" /> 
     <passthru name="Priority"                 input="Priority" />      
     
     <recordEval field="ReadCmdPri" type="auto" defaultVal="0">
         <item value="@Priority" expression="Event == "ReadCmd"" />
     </recordEval>
     
     <passthru name="Event"                    input="Event" /> 
     <passthru name="Size"                     input="Size" />
     
     <recordEval field="TotalSize" type="integer" display="Decimal">
         <item expression="accum(Size)"/>
     </recordEval>
     
     <recordEval field="DeltaTime" type="integer" display="Decimal">
         <item expression="delta(Timestamp)"/>
     </recordEval>
 </translate>


  • Output
Timestamp Priority ReadCmdPri Event Size TotalSize DeltaTime
1000 0 0 ReadCmd 8 8 100
1100 0 ReadResp 8 16 100
1200 2 WriteCmd 8 24 100
1300 3 WriteCmd 8 32 100
1400 1 1 ReadCmd 22 54 100
1500 0 WriteCmd 8 62 100
1600 1 ReadResp 16 78 100
