Time Correlator with special Sync and Start events[edit]

Overview[edit]

This DP is designed to correlate multiple homogenous time based data streams with special synchronized event records. Each data stream has records with its own local clock time value.

At a point of time, all sources will log a special "sync" event simultaneously such as triggered by a system interrupt. With records from the sync event, the local clocks can be correlated into a common timeline. All the sync event records from all data streams should be fed into the 1st special port.

Also, the master channel will specify the start event and this data should be fed into the 2nd special port. The master time line will zero adjusted to this start event.

Input Channel[edit]

Number of input data streams plus 2 (N+2) input channels with minimum 2 data streams.

  • First N channels are data stream
  • The Nth+1 channel is the Time Sync Port
  • The Nth+2 channel is the Start Reference Port

Output Channel[edit]

1 output channel

Properties[edit]

Property Description Type Default
Num Inputs The number of input channels Integer
masterChannel Specify which channel is the master one. If this value remains the default (-1), the channel with the smallest time value will be the reference master channel. Integer -1
Flush When All Channels Have Discontinuity Set this to true false not supported Boolean false
Keep Previous Value Substitutes undefined field values (post merge) with the last valid value Boolean true
Merge Non Primary Merges duplicate fields/columns. When set to false, duplicate fields are not merged and field names are modified to have unique names with the format fieldname.n (n=0,1..). When set to true, the original field name is used Boolean false
Name Displays the unique name for the component. Name can be changed through GSB Text
Ref Field The name of the field based on which the correlation will be done. Text
