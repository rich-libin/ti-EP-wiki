Overview
[edit]

Although often based on well-known communication standards such as RS-232, RS-485, CAN or Ethernet there are very specific and diverse standards available for industrial communications.

Examples based on serial comms:

  • PROFIBUS
  • Modbus
  • CAN

Ethernet based:

  • EtherCAT
  • PROFINET
  • Ethernet/IP
  • SERCOS III
  • PowerLink

Usually each standards requires implementation in hardware and an appropriate communications software (often called 'stack'). Dedicated hardware therefore reduces flexibility of available system solutions.

Solutions[edit]

There are a lot of devices that can be used to implement industrial communication interfaces. Implementing any of the Real-time Ethernet slave protocols requires a PRUSS v2 which is for example available with the Sitara AM335x family of embedded processors. See the category links below for for further information on hardware and software deliverables from TI dedicated to industrial communication.

TI software related to Industrial Communication running mostly on AM335x devices will be released as part of the Industrial Automation SDK.
